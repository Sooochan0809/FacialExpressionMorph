# FacialExpressionMorph

FacialExpressionMorphingは、ブラウザ上で `A` と `B` の各パネルからカメラ撮影またはファイル追加を行い、2枚の顔画像の間をモーフィングするアプリです。  
各パネルのライブ映像には表情推定結果が表示され、保存した時点の感情ラベルもそのまま残ります。

## 概要

このアプリでは、Webカメラ（またはファイル）から取得した顔画像 2 枚をキャプチャし、それぞれの顔ランドマークをもとに中間形状を生成して、スライダーでモーフィング結果を調整できます。

- `A` パネルの `撮影` ボタンで 1 枚目を保存
- `B` パネルの `撮影` ボタンで 2 枚目を保存
- スライダーで A から B への補間率を変更
- `A` / `B` / `Result` の各画像を PNG でダウンロード
- `A` / `B` の各パネルで表情推定結果を表示

## 流れ

1. ページにアクセス(https://sooochan0809.github.io/FacialExpressionMorph/)
2. カメラ使用を許可する
3. 正面に近い状態で顔を映す
4. `A` と `B` の各パネルで、表情を変えながら `撮影` する
5. スライダーを動かして中間表情を確認する

## 使い方

### 1. リポジトリを開く

```bash
git clone <repository-url>
cd FacialExpressionMorphing
```

### 2. ローカルサーバーを起動する

`file://` で直接開くと、カメラやモジュール読み込みがうまく動かないことがあるため、HTTP サーバー経由で起動してください。

例:

```bash
python3 -m http.server 8000
```

### 3. ブラウザでアクセスする

```text
http://localhost:8000
```

初回アクセス時にカメラ使用の許可が求められます。

## 必要環境

- カメラにアクセスできるモダンブラウザ
- ローカル HTTP サーバー
- インターネット接続

補足:

- MediaPipe の `FaceLandmarker` は CDN / 外部モデル URL から読み込まれます
- `face-api.js` 用のモデルは `./models` ディレクトリから読み込まれます

## 技術構成

### フロントエンド

### 利用ライブラリ

- `face-api.js`
  - 顔検出と表情推定
- MediaPipe Tasks Vision `FaceLandmarker`
  - 顔ランドマーク抽出
- `Delaunator`
  - 三角形分割

## 実装の流れ

1. `A` / `B` パネルのカメラ映像を取得する
2. `face-api.js` で表情を推定する
3. A / B の撮影画像をそれぞれのキャンバスへ保存する
4. MediaPipe で顔ランドマークを取得する
5. ランドマークを補間する
6. Delaunay 三角形分割で各領域をワープする
7. A と B をアルファブレンドして結果を描画する

## 動画版

動画フレームどうしのモーフィングを試すページは `video.html` です。

- `index.html`
  - 静止画 / カメラ撮影ベースのモーフィング
- `video-morph.html`
  - A/B の動画ファイルを読み込み、同じ時刻のフレームどうしをリアルタイムでモーフ表示

補足:

- `video-morph.html` は現時点では動画ファイルのリアルタイムプレビュー向けです
- 動画そのものの書き出しではなく、Resultフレームの PNG 保存に対応しています

## 注意点

- 顔が正面から大きく外れるとランドマーク検出に失敗することがあります
- A と B の構図差が大きいほど、モーフィング結果が不自然になることがあります
- HTTPS ではなくローカル HTTP で動かす場合でも、`localhost` であれば多くのブラウザでカメラ利用が可能です
- 外部 CDN に依存しているため、オフライン環境ではそのままでは動作しません
